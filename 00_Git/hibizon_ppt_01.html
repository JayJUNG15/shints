2. 발표 설계 방향

  • 발표는 계획서 순서가 아니라 평가항목 순서와 심사위원 이해 순서에 맞춰 재구성함
  • 다만 슬라이드 내용은 모두 실제 제출본 문구와 구조에서 가져오는 방식으로 정리함
  • 발표 초반에는 왜 필요한가, 무엇을 하는가, 무엇이 다른가를 빠르게 이해시키고
  • 중반에는 어떻게 추진하는가, 누가 하는가, 무엇으로 검증하는가를 보여주고
  • 후반에는 어디에 쓰이는가, 왜 실행 가능한가, 예산과 장비가 맞는가를 정리하는 흐름으로 구성

5. RFP 대응 표시 방식

  • 중요 장표 우측 상단에 RFP 대응 또는 계획서 반영 포인트 박스를 넣는 것을 권장
  • 이 박스에는 안전의복, AI, 실증, 교육, 보급, 사업화 중 해당 장표 핵심만 2~4개 기입
  • 심사위원이 장표를 넘길 때마다 이 내용이 무엇을 설명하는 장표인지 즉시 보이게 만드는 것이 중요
